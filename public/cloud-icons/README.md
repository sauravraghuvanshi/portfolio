# Cloud Icons

This directory contains SVG icons for the Architecture Playground (`/playground`).

## Layout

```
public/cloud-icons/
├── azure/
│   ├── compute/         # app-service.svg, functions.svg, vm.svg, …
│   ├── storage/
│   ├── database/
│   ├── networking/
│   ├── ai/
│   ├── identity/
│   ├── integration/
│   └── monitor/
├── aws/                 # same category structure
└── gcp/                 # same category structure
```

The build script walks this tree and emits `content/cloud-icons.json`, which
the playground reads to populate its palette.

**Filename = slug.** `app-service.svg` becomes the slug `app-service` with the
auto-generated label `App Service`. Hyphenate multi-word service names.

## Adding a new icon

1. Drop the SVG in the right cloud + category folder.
2. Run `npm run build:icon-manifest`.
3. The new icon appears in the palette on next dev refresh / build.

## Expanding to the full official sets

The repo ships with ~100 minimalist starter icons (clean brand-colored shapes)
so the playground works out of the box. To swap to the full official sets:

| Cloud | Source | Approximate count |
|---|---|---|
| **Azure** | <https://learn.microsoft.com/en-us/azure/architecture/icons/> — download "Azure Architecture Icons" | ~600 |
| **AWS** | <https://aws.amazon.com/architecture/icons/> — download "AWS Architecture Icons (SVG)" | ~500 |
| **Google Cloud** | <https://cloud.google.com/icons> — download official icon set | ~300 |

After downloading:

1. Extract the SVGs.
2. Sort them into the matching `<cloud>/<category>/` folders.
3. Rename to `kebab-case.svg`.
4. Delete the matching starter SVG to replace it.
5. Run `npm run build:icon-manifest`.
